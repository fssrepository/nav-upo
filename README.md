# E-Kozig

A legtöbb közigazgatási projektet – pl. a NAV Ügyfélportál – zárt körülmények között fejlesztik teljes, integrált környezettel együtt (felhasználói felület, adatbázis, szerverinfrastruktúra stb.), ami már késöbb nehezen és jelentős költséggel módosítható, ha a felhasználók elégedetlenek a végeredménnyel.

Ha egy projekt felületi prototípusa már az integrált fejlesztési szakasz előtt a felhasználók elé kerül, a problémák sokkal korábban és gyorsabban azonosíthatók, ami akár 80% költségcsökkentést is jelenthet. Ezek javítása ebben a fázisban jellemzően 1–2 órát vesz igénybe, szemben a későbbi, akár hónapokig tartó módosításokkal.

Miután kialakul egy konszenzus arról, hogy a felhasználók milyen működést és funkciókat várnak el, pl. a NAV ezeket az elvárásokat egyértelműen át tudja vezetni a háttérfolyamatokba. Ennek eredményeként a felhasználók akár 6 hónappal előre fel tudnak készülni a várható változásokra. A GitHub integrált együttműködési környezetet biztosít, ahol vitafórumok hozhatók létre, külsős és NAV-os fejlesztők egyaránt bevonhatók, valamint megfelelő – alap szintű – képzéssel akár a könyvelői kamara is részt vehet a rendszer karbantartásában.
